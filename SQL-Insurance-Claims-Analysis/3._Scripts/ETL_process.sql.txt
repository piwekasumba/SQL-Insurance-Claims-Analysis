Commit – Basic ETL and Data Transformation Using SQL

This stage focuses on applying basic ETL (Extract, Transform, Load) concepts using SQL within the insurance claims dataset.

The goal is to understand how data can be prepared for analysis by combining extraction, simple transformations, and structured loading into usable tables.

At this stage, the following was done:

- Extracted data from existing tables for further processing
- Applied basic cleaning steps such as handling missing values and checking for duplicates
- Created simple transformations, including calculated fields and derived values
- Structured queries to prepare data in a more analysis-ready format

Key focus during this stage:

- Understanding how data changes during transformation
- Observing how small adjustments affect final outputs
- Maintaining consistency when preparing data across multiple tables

• This stage builds on earlier steps and introduces a more structured way of preparing data before analysis, while still focusing on learning core SQL concepts.
