Commit – Data Loading Stage

This stage focuses on loading data into the previously created tables for the insurance claims dataset.

The goal here is to ensure that data is inserted correctly and is ready for further analysis using SQL queries.

At this stage, the following was done:

- Inserted data into the defined tables
- Verified that records were loaded correctly
- Checked for basic issues such as missing or inconsistent values

This step is important because:

- Data quality at this stage affects all downstream analysis
- Incorrect or incomplete data can lead to misleading results
- Proper loading ensures consistency when working across multiple tables

• This forms the foundation for the next stage, which involves querying and analyzing the data using joins and other SQL operations.
