Commit – Data Analysis with SQL Queries

This stage focuses on analyzing insurance claims data using SQL queries.

The goal is to move beyond basic queries and start working with data across multiple related tables, while paying attention to how query structure affects results.

At this stage, the following was done:

- Wrote SQL queries to explore claims and related data
- Used joins to combine data from multiple tables
- Applied filters and aggregations to answer specific questions
- Checked results for consistency across different queries

Key focus during this stage:

- Understanding how joins affect outputs (duplicates, missing data)
- Ensuring results remain consistent when queries are adjusted
- Structuring queries in a clear and readable way

• This stage builds on the data setup and prepares for more refined analysis, where accuracy and consistency become more important than just getting a result.
